Tutorial 2: Hello, World!

You set up a Yellow Spreadsheet for the first time. You are now a power user!

  1. Copy our template to your Google Drive account.
  2. (Say yes to "Make a copy?")
  3. You should be in the Hello World tab. Some of the cells are bright yellow. We chose the same shade of yellow that you've seen in Word docs where you're expected to fill in the blanks.
  4. Go ahead and fill in your particulars in the blank Row 14. If you leave it blank, we'll make something up for you. Change the email address for the Acme Example company to be one of your email addresses.
  5. What about the cells that aren't in yellow? You can look, but don't touch. Yet.
  6. Create an agreement! Click Add-ons / Legalese / Generate PDFs. Approve the authorization request.
  7. A link should appear in cell E6 of the spreadsheet. It leads to an output folder in Google Drive.
  8. In that folder, you should see an XML file. Yay! You are not expected to understand the XML. That's the job of the Legalese Robot.
  9. Wait three minutes for the Legalese Robot to turn the XML into a PDF.
  10. (While you wait, you might check out the README file.)
  11. If, after three minutes, a PDF doesn't appear, just reload the web page.

After running this tutorial, you will have: gotten comfortable with Legalese's Yellow Spreadsheets, created a PDF of your own, read the README, and glanced at the XML source files.

I noticed the output folder was shared with robot@legalese.io. What if I have confidential information? The Legalese Robot operates without any human involvement, turning XML into PDF tirelessly, day and night. In the normal course of operation, no human views your agreements. If debugging or troubleshooting is needed, only with your permission will a human view your XML or PDF files. (Errors are logged to a separate file which help us debug without ever viewing your agreements.)

Are the yellow spreadsheets shared? No, the yellow spreadsheets are not shared with the Legalese Robot. The robot only has access to your XML, which it needs to generate the PDFs.

Tutorial 3: Send for Signature

You, a power user, inject the Hello World PDF into an e-signature service which emails it to you for signing.

  1. Open the Google Drive folder that contains the Hello World PDF.
    • The folder is named My Legalese Tutorial Hello World.
    • (If you don't know how to find the folder, begin by going back to the yellow spreadsheet.)
    • (If you don't know how to find the yellow spreadsheet, search your Google Drive for My Legalese Tutorial.)
    • (If you don't want to search, start at My Drive and open Legalese Root. Everything that Legalese does is in there. We created this folder for you automatically.)
  2. Select the Hello World PDF.
  3. Right-click the PDF (or hit a if you're a keyboard geek).
  4. Open With: E-Sign with EchoSign.
  5. If you don't see the EchoSign option, click Connect More Apps to add it.
  6. Enter the To: and CC: addresses based on what's shown in the README file. The order matters.
  7. If you're using EchoSign, you'll notice that the signature blocks are already in the right place.
  8. Go ahead and send for signature!

After running this tutorial, you will have: set up an e-signature back-end and sent a PDF for signature.

When the signature service asks me for the signers' email addresses, what should I say? Open up the README file and enter the email addresses shown in the To: line.

What if I prefer DocuSign / HelloSign / some other e-signature backend? That's fine; you can certainly do that. The advantage of EchoSign is that the PDFs contain embedded Signature Blocks which EchoSign knows how to read. If you use some other signature service, you will need to manually place the signature blocks.

Tutorial 4: A Real Nondisclosure Agreement

You, a power user, choose a real-world docset, and create a real-world agreement, taking advantage of the INCLUDE keyword to reuse the entities from a different tab.

  1. Look at the row of tabs at the bottom of the spreadsheet.
  2. You will see tabs with names like NDA, Entities, Incorporation, Class F, and Seed Round.
  3. Let's start with a simple agreement: a nondisclosure.
  4. Go to the NDA tab.
  5. Note that one of the section headings is INCLUDE Hello World. That tells Legalese to load the Entities data from the Hello World tab.
  6. Generate PDFs and wait a couple minutes for the PDF to show up.
  7. This time, you should see that the PDF is now a nondisclosure agreement.
  8. There are two kinds of NDAs: unilateral and bilateral. Change cell B9 to suit, and generate again, if necessary.
  9. If you actually want to send a real NDA, update the Entities in Hello Word with real particulars, then repeat the steps from Tutorial 3.

After running this tutorial, you will have: created a real NDA ready for execution.

What does the Counterparty line in the ROLES section mean? The NDA template uses two parties: the Company, and the Counterparty. Cell B13 says [Buyer], which means: every Buyer is a Counterparty, so Legalese defines Counterparties by substituting the Buyer from the Hello World tab.

Tutorial 5: Introducing Docsets

You, a power user, browse all the docsets that are available in Legalese. A docset generates one or more agreements. So far, all the docsets you've seen contain only one agreement.

  1. Look at the row of tabs at the bottom of the spreadsheet.
  2. You will see tabs with names like NDA, Entities, Incorporation, Class F, and Seed Round.
  3. You can generate dummy agreements by going to each tab and clicking Generate PDFs. Each tab except Entities, that is.

Entities

This tab is special. It contains the particulars of the parties to your agreements. It doesn't generate any agreements; other agreements refer to this.

Why? Because the names and addresses of your company, founders, employees, and investors are details used by multiple different agreements. The DRY principle says we should keep all that information in one place and not repeat it. So it lives in the Entities tab instead of repeating across multiple tabs.

Incorporation

All the paperwork involved in an incorporation is available here:

  • Memorandum and Articles of Assocation
  • Form 45 (if Singapore)
  • JFDI Standard Shareholders Agreement
  • JFDI Standard Volunteer Agreement

In the next tutorial, you produce all the paperwork needed for incorporation.

  3. In a few minutes you should see dummy documents for Frog Porridge Pte. Ltd., a company with three froggy founders and one wormy intern.
  4. Look under the Configuration section. You'll see a line that says Templates:, followed by four templates named jfdi_memorandum, form45, jfdi_shareholders_agreement, and jfdi_volunteer_agreement. When you click Generate PDFs, those templates get filled.

Tutorial 7: Developing your Own Templates

You, a content developer, tweak an existing template for your own use, using Github. This involves learning a little bit of XML. You publish your template publicly. You instruct Legalese to use your template.

Legalese has a large number of standard available templates. They are listed in the Available Templates tab of the Legalese Demo Master spreadsheet. Every time you run Generate PDFs, Legalese reads this tab. If it didn't, Legalese wouldn't know where to load the templates that you specified under the CONFIGURATION section.

You can create new XML templates of your own, and serve them at URLs of your choice. But how do you tell Legalese about those new templates?

Easy. Just set up an AVAILABLE TEMPLATES section in your own spreadsheet! INCLUDE it. But also INCLUDE the regular one, too.

  1. These instructions assume you have created an XML template of your own devising, and published it at some public URL. The simplest way to do this is to download the XML source of one of the existing templates, and republish it on the web at your own personal website, if you have one.
  2. To see how this all comes out, take a look at the spreadsheet called Dev Hello World.
  3. Start by viewing the Legalese's master Available Templates tab.
  4. Copy to your spreadsheet. Call the new tab My Templates.
  5. Delete all the existing template definitions. Well, all but the one example that you copied from. Or you could put ignore: into Column A, which has the same effect.
  6. Give a new name to your template. It could be my_test_hello_world.
  7. Set the URL for your template.
  8. Your URL has to be accessible by anyone on the Net. Try loading that URL with an Incognito Mode window. If that doesn't work, Legalese won't be able to retrieve your template.
  9. Set the To and CC fields to be the parties that will be signing the agreement.
  10. Set Nocache=1. If you don't, your development cycle may be slowed, because by default Legalese caches all fetches for 5 minutes.
  11. Back in your terms spreadsheet, under CONFIGURATION, name your template under the Templates: config.
  12. Also in your terms spreadsheet, all the way under the CONFIGURATION section, add two INCLUDE lines:
    • INCLUDE https://docs.google.com/spreadsheets/d/1rBuKOWSqRE7QgKgF6uVWR9www4LoLho4UjOCHPQplhw/edit#gid=981127052
    • INCLUDE My Templates
  13. You can see an example at the bottom of the Seed Round tab.
  14. That should be a literal string, My Templates, or whatever you named your new tab.
  15. Run Generate PDFs to see your template in action!

If there's a problem, you will need to debug. go into Tools / Script Editor. An IDE tab will open. View / Execution Transcript and View / Logs will help you trace the problem.

What if I want to publish my template privately? You should be able to do Basic HTTP Authentication and embed a username and password into the URL. You should of course use HTTPS. We haven't tried this yet, but it should work.
